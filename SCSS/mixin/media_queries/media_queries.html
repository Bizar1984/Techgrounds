Media queries

Media Queries with SCSS @supports; allows to use expressions in declaration queries. For example: @mixin sticky-position {
{ position: fixed;
@supports (position: sticky)
}
.banner {
@include sticky-position;
};

Using variables and @content. Check the example:
@mixin respond-to($media) {
@if $media == handhelds {
@media only screen and (max-width: 479px) { @content; }
} @else if $media == wide-handhelds {
@media only screen and (min-width: 480px) and (max-width: 767px) { @content; }
} @else if $media == tablets {
@media only screen and (min-width: 768px) and (max-width: 959px) { @content; }
}
}

#sidebar {
float: left;
width: 300px;
@include respond-to(handhelds) { float: none; }
@include respond-to(wide-handhelds) { float: none; }
@include respond-to(tablets) { width: 240px; }
}

Variables
$information-phone: "only screen and (max-width : 320px)";
$extra-small: "only screen and (max-width : 576px)";
$small: "screen and (max-width: 768px) and (min-width: 576px)"
$medium: "screen and (max-width: 992px) and (min-width: 768px)"
$large: "screen and (max-width: 1200px) and (min-width: 992px)"
$extra-large: "only screen and (min-width : 1200px)";
$landscape:: "only screen and (orientation: landscape)";
@media #{$information-phone} {
background: red;
}

More
$min-width-name: min-device-width;
$max-width-name: max-device-width;
$target-width: 320px;

@media screen and ($max-width-name: $target-width) {
color: white;
}

Increasing the max width for a new breakpoint
@media screen and ($max-width-name: $target-width + 20) {
color: white;
}

@extend and @mixin. Which one to use?

@extend is used more for shared, static and similar styles. It allows you to share styles between selectors,
analogue to div class= "error newError"
@mixins for dynamic styles with passed arguments which are leading to
repeated declarations. No-repeat please...
%: called a plaeholder selector, this is NOT compiled

Some examples: @mixin error {
border: 1px solid black;
background-color: #fdd;

.newError {
@include error;
}

.notSoSeriousError {
@include error;
}
}

Above compiles to the following:
.newError {
border: 1px solid black;
background-color: #fdd;
}
.notSoSeriousError {
border: 1px solid black;
background-color: #fdd; }

Declarations are being repeated, this is bad practice. Try to pass in parameters.

Composes(CSS- Modules) leads to duplicate classes, but appear in the markup, not in the stylesheets
